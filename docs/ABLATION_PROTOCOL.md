# Ablation Protocol (Reviewer Must-Fix)

This ablation isolates core contribution components in the requested order:

1. `regression` (baseline detector outputs)
2. `+control` (control-payload / confidence filter)
3. `+delta_ratio/IF` (ML anomaly scoring threshold)
4. `+OOB` (callback confirmation for BXSS/SSRF)

## Stage Definitions
- **regression**: all raw module findings considered predicted positives.
- **control**: retain findings with evidence/confidence gates (`MEDIUM/HIGH`, confirmations, or explicit vulnerable flags).
- **delta_if**: apply ML filter (`anomaly_score=-1` OR `ml_score >= threshold`, default 0.55).
- **oob**:
  - SSRF: require `confirmed=true`.
  - BXSS: require correlation/callback indicators (`callback_timestamp` or `delay_seconds`).
  - SQLi/CMDi/XXE: same as `delta_if` (non-OOB classes).

Artifacts include:

## Interpretation Guidance
- Improvement from `regression` -> `control` quantifies false-positive suppression from control payload logic.
- Improvement from `control` -> `delta_if` quantifies ML contribution.
- Improvement from `delta_if` -> `oob` quantifies confirmation-strength contribution for OOB classes.
